### Mecanismos

**#1 — `visibleWidth` pós-normalização** (`utils.ts`): toda linha renderizada carrega o
sufixo `SEGMENT_RESET` (contém ESC), então os fast paths ASCII/Latin — que testam a
string *crua* — falhavam em 100% das linhas do frame, e cada linha pagava
segmentação de graphemes completa (~11–27 µs/linha). O patch re-testa os fast paths
na string *limpa* (após strip de ANSI e expansão de tabs): largura = `clean.length`
pelo mesmo argumento de correção de `isLatinFastPathEligible`. Afeta full render
(resize, primeiro paint), compositing de overlays, wrap e medição de tabelas markdown.

**#2 — `collectKittyImageIds` memoizado** (`tui.ts`): em terminais Kitty, o walk de
IDs de imagem rodava `indexOf("\x1b_G")` sobre o frame inteiro a cada commit. Linhas
inalteradas chegam como os mesmos objetos string (contrato de memoização dos
componentes — o mesmo que `resetInputCache` já usa), então os IDs extraídos são
reutilizados por comparação de ponteiro indexada contra o último frame commitado.
Dois arrays por linha alternam papéis a cada commit (zero alocação em steady state).
